HashMap的实现原理

简单来说，HashMap对数据的存储是把key和value封装为一个对象也就是Entry进行存储的，每一个Entry都会存储在Entry[]中

  • 在进行数据存储的过程中，对需要插入的Entry计算Hash值，根据Hash值计算其在Entry[]中的存储位置，再根据Equals方法获取其在链表中的位置

  • 当需要取出一个Entry时，也会根据hash算法计算其在其在数组中的位置，再根据Equals方法在链表中取出该Equals

SpringBoot的自动装配原理

SpringBoot在启动的时候会通过@EnableAutoConfiguration注解找到META-INF/spring.factories配置文件中的所有自动配置类，并对其进行加载，这些自动配置类都是以AutoConfiguration结尾来命名的，它实际上就是一个JavaConfig形式的Spring容器配置类，通过@Bean导入到Spring容器中，以Properties结尾命名的类是和配置文件进行绑定的。他能通过这些以Properties结尾命名的类中取得全局文件中配置的属性，我们可以通过修改配置文件对应的属性来修改自动配置的默认值，来完成自定义配置

JVM的组成

  1. 类加载器(ClassLoader)
  2. 运行时数据区(Runtime Data Area)
  3. 执行引擎(Execution Engine)
  4. 本地库接口(Native Interface)

程序在执行之前先要把Java代码转化为字节码(class文件)，jvm首先需要把字节码通过一定的方式-类加载器(ClassLoader)-把文件加载到内存中运行时数据区(Runtime Data Area)，而字节码文件是JVM的一套指令集规范，并不能直接交给底层操作系统去执行，因此需要特定的命令解析器执行引擎(Execution Engine)将字节码翻译成底层系统指令再交由CPU去执行，而这个过程中需要调用其他语言的接口也就是本地库文件(Native Interface)来实现整个程序的功能。

GET和POST的区别

比较项 GET POST
后退刷新 无害 数据会被重新提交
书签 可收藏为书签 不可收藏为书签
缓存 能被缓存 不能被缓存
历史 参数保留在浏览器历史中 参数不会保存在浏览器历史中
对数据长度的限制 URL的最大长度是2048个字符 无限制
对数据类型的限制 ASCII字符 无限制
安全性 由于是在URL明文对数据进行发送，所以安全性是相对较差的 参数不会保存在浏览器历史或者web服务器日志中
可见性 URL中可见 URL中不可见

这样来说，GET和POST的区别如下：

  • GET用于获取信息，是无副作用的，是幂等性，且可以缓存
  • POST用于修改服务器上的数据，有副作用，非幂等，不可缓存

Redis的数据类型

  • string 二进制安全的字符串
  • Lists 按插入值排序的字符串元素的集合。他们基本上就是链表
  • Sets 不重复且无序的字符串元素的集合
  • Sorted sets 类似Sets，但是每个字符串元素都关联到一个叫score浮动数值
  • Hashes 由field和关联的value组成的map
  • Bit Arrays ton过特殊的命令，可以将string值当做一系列可以设置和清除的bits
  • HyperLogLogs 这是一个被用于估计一个set中匀速数量的概率性的数据结构
  • Stream 5.0版本新增，可以用来做持久化的消息队列

在浏览器中输入URL后，执行的全部过程

  • 浏览器向DNS服务器发送DNS查询请求 -> 完成域名到IP地址的转换
  • 客户端向web服务器发送HTTP请求
    • HTTP请求是一个基于TCP协议之上的应用层协议，所以需要建立TCP连接，三次握手
  • 服务器给浏览器进行一个302永久重定向响应
  • 发送相应数据给客户端
  • 浏览器对服务器的相应数据进行响应
    • 浏览器响应服务器相应的资源，会对资源进行一些缓存，缓存完成之后浏览器会关闭TCP连接
