封装、继承、多态(、抽象)

什么是面向对象的编程思想

  • 程序员从面向对象的执行者变为面向对象的指挥者
  • 万事万物皆对象

如何里理解万事万物皆对象？

  1. 在Java语言的范畴中，我们将功能、结构等封装到类中，通过类的实例化，来调用具体的功能
  2. 涉及到Java语言和前端HTML、后端的数据库进行交互的时候，前后端的结构在Java层面交互时，都体现为类、对象

面向对象和面向过程的区别

  • 面向过程强调的是功能行为，以函数为最小单位，考虑怎么做
  • 面向对象强调具备了功能的对象，以类、对象为基本单位，考虑谁来做

面向对象和面向过程优缺点

  • 面向过程
    • 优点：性能比面向对象编程效率高，类调用需要实例化，开销比较大
    • 缺点：没有面向对象容易维护，易扩展，易复用
  • 面向对象
    • 优点：
    • 缺点：

面向对象的两个要素

类：对一类事物的描述，是抽象的、概念上的定义

对象：是实际存在的该类食物事物的每个个体，因而也被成为实例

面向对象设计的重点是类的设计，类的设计，其实就是类的成员的设计

Java类以及类的成员

属性 = 成员变量 = field = 域、字段 在Java核心技术卷中，表示类成员属性表示的是静态域

方法 = 成员方法 = 函数 = method

创建类的对象 = 类的实例化 = 实例化类

类和对象

类和对象的使用

  1. 创建一个类，设计类的成员
  2. 创建类的对象
  3. 通过“对象.属性”或者“对象.方法”调用类的结构

一般情况下，我们不建议对类的属性进行对外开放，也就是public修饰，因为这样就破坏了我们封装性。最好的办法就是使用getter和setter的方法，这个这两类方法实现对类中属性进行更改，或者对于常量，直接使用final关键词不允许修改。

一个类的多个对象之间的关系

如果创建了一个类的多个对象，则每个对象都独立的拥有一套类的属性。需要注意的是，这些属性都是非static的。如果修改一个对象的属性a，则另外一个对象的属性a不受影响。

对象的创建和使用的内存解析

  • 堆：存放对象实例

  • 栈：存储局部变量、对象应用

  • 方法区：存储被JVM加载的类信息，常量，静态变量，解释编译器编译后的代码

引用类型变量，只可能存储两类值：null或者地址值

类中属性的使用

属性和局部变量的区别

  • 相同点

    • 定义变量的格式：数据类型 变量名 = 变量值
    • 先声明，后使用
    • 变量都有对应的作用域

  • 不同点

    • 位置不同
      • 属性直接定义在类的{}中
      • 局部变量声明在方法内、方法形参、代码块内、构造器形参，构造器内部
    • 关于权限修饰符的不同
      • 属性可以使用常见的权限修饰符来修饰
      • 局部变量不可以使用权限修饰符
    • 默认初始化值的情况
      • 属性：类的属性，根据其类型，都有默认的初始化值
      • 局部变量：没有默认的初始化值
        • 特别的，形参在调用时，我们赋值即可
    • 在内存中加载的位置也是不同的
      • 非static属性：加载到堆空间中
      • 局部变量：加载到栈空间中

常见的权限修饰符：private、public、protected、缺省

权限大小：public > protected > private

类中方法的声明和使用

方法：描述类应该具有的功能

方法可以有形参，也可以没有形参；可以有返回值，也可以没有返回值

方法的声明

权限修饰符 返回值类型 方法名(形参列表){ 方法体 }

static、final、abstract也可以修饰方法

说明
  • 关于权限修饰符，默认的都是使用public
  • 返回值类型
    • 如果有返回值，那么必须在方法声明时，指定返回值的类型。与此同时，方法中需要使用return关键字来返回指定的类型变量或者常量
    • 没有返回值，方法声明时，返回类型为void，可以没有return，但是如果要是用return，那么就是以下的结构return;，只起到控制方法结束的作用
  • 什么时候该使用返回值
    • 看看我调用方法是否需要对原本的值进行一个反馈
  • 方法名
    • 遵循命名规则，见名知意
  • 形式参数
    • 就可以有0个、一个或者多个形式参数
    • 格式：数据类型1 形参1，数据类型2 形参二，$\cdots$​
    • 是否需要设置形参（凭借经验，具体问题具体分析）
  • 方法体：方法功能的体现
return关键字的使用

作用范围：使用在方法体中

作用：

  1. 结束方法
  2. 针对有返回类型的方法，使用return 数据的方法返回所需要的数据
  3. return关键字后面不可以声明执行语句
方法的使用

方法的使用过程中，可以调用类的属性，也可以调用类的方法。需要注意的是，static修饰的方法只能直接调用static修饰的方法，否则就需要先实例化类，然后通过对象调用。

特殊的，如果方法调用了它本身，这就是递归。递归的使用能够极大地使代码整洁，但不是所有的方法都可以写成递归，因为递归是需要边界判断的条件，如果没有边界，那么就违背了算法的特征。

方法中不可以定义方法

一个例子

在StudentTest类中，对对象数组进行赋值的时候，出现了一些问题，但这当我使用foreach对对象数组进行遍历的时候，出现了很多问题。首先就是对象数组进行初始化的过程，由于没有对对象数组中的对象进行初始化，也就是说，每一个依然是null。尽管我对s进行了一个初始化操作，但是，s本身就是对students[i]的引用，相当于是students[i]的复制，所以使用foreach就会出现问题使用set并没有将对象存入数组。

或者说的更加专业一些，foreach循环不能给数组赋值。每一次loop实际只是获得了当前元素的值，却没有获得当前元素在数组中的对象句柄。这样就无法直接操作对应的句柄。

当然，你也可以理解为foreach相当于是先对对象数组进行了一个复制，其实本质上，foreach是通过迭代器实现的，但是并没有实现复制操作，所以性能不会很好。

总结：

在普通array、arraylist这种连续内存存储的结构中，遍历结构的性能：for循环 > 迭代器 > foreach。
在linklist这种不连续结构中，遍历结构的性能：迭代器 > foreach >> for循环。

匿名对象的使用

  1. 理解：创建的对象，并没有显式地赋值给一个变量名，例如：new Student().getScore()
  2. 特征：匿名对象，只能调用一次
