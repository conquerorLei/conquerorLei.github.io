markdown语法以及Typora使用(基础)

发表于 2021-06-13 更新于 2021-08-21 分类于 Software ， Typora 阅读次数：
本文字数： 3.2k 阅读时长 ≈ 3 分钟

前言

由于程序员的工作性质以及行业的变化特点，我们需要不断地学习新知识补充新知识以适应公司的要求，可能需要做大量的笔记。电脑做笔记是很有必要的。我个人很喜欢使用markdown语法，便捷以及高效，以下是对markdown常用语法的一些终结，markdown高级用法，将在下一篇展示。

简介

由于Markdown的轻量化、易读易写特性，并且对于图片，图表、数学式都有支持，许多网站都广泛使用Markdown来撰写帮助文档或是用于论坛上发表消息。 例如程序员经常使用的Github以及gitee都建议使用md文档进行初始化或者说明仓库用途。

编辑Markdown的免费软件有很多，我偏好使用的是Typora，下载地址，客户端支持实时预览，其他的编辑器我没有怎么用过。

下面我会结合Typora快捷键说明Markdown语法

基础语法

¶标题

官网在标题的设定上，提供了两种形式，Setext和atx，以下在代码块中展示

¶Setext

一级标题是以"====“作为底线，二级标题是以”—"作为底线

两个需要注意的点：

  1. 无论底线为"=“还是”-"，无论数量为多少个都会有效果，不用卡数量
  2. 底线行不能有其他的字符

¶atx

设置标题，只需要在标题前加上"#"就好了

需要注意一下两点：

  1. 需要设置为几级标题就添加几个"#"
  2. 一共有六级标题，也就是说最多只能设置六级标题
  3. 最后一个"#"和标题首字符之间有一个空格，否则就会不适用

相比Setext，atx使用简直太方便了

¶Typora快捷键

Typora支持atx以及Setext，默认情况下使用的是atx，使用快捷键能够更快地设置标题

快捷键 说明
Ctrl+1 设置一级标题
Ctrl+2 设置二级标题
Ctrl+3 设置三级标题
Ctrl+4 设置四级标题
Ctrl+5 设置五级标题
Ctrl+6 设置六级标题

若要更改Typora配置使用Setext需要更改偏好设置。

😥对不起，我可能有浪费空间的嫌疑​

Ctrl+n == 设置n级标题 n为0-6之间的整数，包括0和6

需要注意两点：

  1. 无论是行内代码还是多行代码，都是反引号

¶任务列表

效果：

  • [ ] 写博客

  • [ ] 输出倒逼输入

  • [x] 写博客

需要注意：

  1. 这里没有快捷键
  2. 注意空格的位置
  3. Typora会自动把’-‘识别为无序列表，在’-'后面加括号，在方阔号内部加空格，在方括号后面加空格

¶Typora快捷键

快捷键 描述
Ctrl+Shift+反引号 行内代码
Ctrl+Alt+F 多行代码

这边就不用符号展示了，渲染出来不好看，所以我直接写汉字了

反引号在键盘左上角TAB键的上面😉

¶有序列表

效果：

  1. 有序列表
    1. 有序列表
    2. 有序列表

以下两点需要注意😉

  1. 无序列表有三种表达方式，‘+’ == ‘-’ == ‘*’
  2. 无论是有序列表还是无需列表都可多级展示，加个Tab就好
  3. 和atx设置标题一样，都需要空格

¶Typora快捷键

快捷键 说明 备注
Ctrl+Shift+{ 有序列表
Ctrl+Shift+} 无序列表
Ctrl+} 缩进 Tab
Ctrl+{ 取消缩进 Shift+Tab

¶链接

效果：

example

注意点总结：

  1. 外部链接没有什么好说的，格式就是那样
  2. 内部链接，链接到的是此md文档所在的目录，支持使用相对路径
  3. 内部链接还可以嵌套，这是为了文章更加美观，其中被嵌套的部分可以是文章的任意部分，可以当做注释来写，多美
